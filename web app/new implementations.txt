Build a production-ready **Token Launch + Swap** feature inside the existing **Axyrion** Solana web app.

## Core context
Axyrion is a Solana memecoin sniper / trading web app. We are adding a **Create Coin** flow directly inside the **Feed** section, plus a **Swap** feature for SPL tokens. The app is already a web app and should remain **mobile-first**, fast, clean, and premium. Avoid generic AI-looking UI. No cheesy gradients, no overdone glassmorphism, no clutter.

The design language should feel:
- sharp
- dark
- premium
- Solana-native
- performance-focused
- minimal but powerful

Use:
- React + Vite + TypeScript
- Tailwind CSS
- Solana Wallet Adapter
- Jupiter swap integration for token swaps / routing
- Supabase for database + storage + launch history
- clean modular architecture
- reusable hooks and components
- proper loading, error, and success states

# FEATURE 1: CREATE COIN FLOW

## Product goal
Inside the **Feed** section, add a visible **Create Coin** button.
When the user clicks it, a modal opens with **two provider options**:
1. **Launch on Bags**
2. **Launch on Pump.fun**

The user selects one provider, fills the token creation form, signs with wallet, submits, and after successful creation:
- token metadata is stored
- launch transaction info is stored in Supabase
- created token appears in local UI immediately
- token becomes part of Axyrion’s tracked ecosystem
- token should be tradeable through Jupiter / supported routing once available

Important:
- build this with a **provider adapter architecture**
- the UI should not be hardcoded only for one provider
- create a common launch interface that both Bags and Pump adapters implement
